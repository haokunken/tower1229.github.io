关于

概述

front-flow-cli是一个基于frontend框架的前端自动化工具，在继承frontend特性基础上，着重提高前端开发自动化程度，实现了项目自动初始化、前端模板引入、less编译、静态资源合并、图片压缩等常见前端需求。

front-flow项目结构

front-flow项目中以下划线开头的目录为编译资源，开发完成后不需要部署到服务器

       
  |--_component/                //UI组件
  |-- _src/                     //开发目录
  |    |-- css/
  |    |-- font/
  |    |-- img/
  |    |-- include/
  |    |-- js/
  |    |-- lib/
  |    |-- index.html
  |    |-- seajs.config.js
  |    `-- favicon.ico 
  |-- modules/                  //seajs插件库
  `-- dist/                     //编译目录

使用

安装

安装nodejs

npm全局安装：npm install front-flow-cli -g

快速开始

0. 新建项目目录，在资源管理器地址栏输入cmd回车打开命令行工具

1. 初始化项目：flow init

2. 启动开发服务：flow run

3. 打包编译：flow build

UI

UI被拆分成多个独立的组件，在_src/css/style.less中配置使用，可以只选择需要的组件

组件配置

全局UI变量均在_src/css/config.less中配置，UI组件原则上都会使用该配置，开发中也应该默认使用配置清单中的变量，使项目风格便于统一调整

组件列表

btn.less 按钮组件，包含普通按钮，幽灵按钮，链接按钮
color.less 色彩组件，包含文字颜色，背景颜色
form.less 表单组件
grid.less 栅格组件
iconfont.less 图标字体组件，依赖_src/font/
iconsprite.less 精灵小图标组件
label.less 标签组件
reset.less 浏览器样式重置
table.less 表格组件
tool.less 常用工具库

编译过程

css编译包括合并文件、less编译、auto prefix三个过程

css引入

css中可以通过/*=include header/style.less */语法引入_src/include下的css文件
