概述

定位

Frontend是面向展示型网站的前端开发框架，致力于提升前端开发效率，并保留最大的页面定制性，因此与“快速搭建”类框架有很大不同，框架css部分只包含复用性较强的基础组件，项目中如果有大量定制化样式，都需要开发者自己编写，这种设定主要为了便于风格化定制，但效率方面提升有限，因此框架另有一个配套的组件库项目，提供自定义组件的开发、演示、调试等功能，可以有效弥补框架UI的缺失；框架js部分内置了一套基于Seajs 的模块库，封装了很多展示型网站常用的功能/效果，对于js模块化增加请求数的问题，框架也内置了本地存储方案来解决。另外，为满足集群建站需求（一个主域名下多个子项目），框架支持多项目并行，各子项目资源独立，互不干扰。

目录结构

整站前端代码都在/static/目录下，./modules/和./ui/分别是模块库和UI，他们构成框架的主体，路径不可更改，其余文件夹均为独立的前端项目，路径可随意更改，其中./web/、./mobile/和./wei/是内置的项目模板，分别用于响应式/PC网站、移动端网站、微画册。

       
  |--Doc/                   //框架文档
  |--static/             //前端静态文件
  |    |-- modules/            //js模块库
  |    |-- ui/                 //css、栅格、图标
  |    `-- favicon.ico 
  |-- mobile/                //移动端项目【模板】
  |-- wei/                  //微画册【模板】
  |-- web/                  //Pc、多终端项目【模板】
  `--test/                  //文档演示用数据

开发环境

框架默认采用绝对路径，开发中需要配置服务器环境，以IIS为例，建立站点后需要填写名称、网站文件地址、端口号，并检查以下设置：

  1. 开启目录浏览
  2. 默认文档添加，如：index.html（按需）
  3. MIME类型添加，常用：json=application/json，woff=application/x-font-woff

其他配置

每个项目根目录下都有一个seajs.config.js文件，这里包含seajs运行相关的所有配置，以下简称配置文件。

网站根路径

seajs.root定义了seajs加载的起始路径，这个路径决定了所有js文件能否正确加载，默认空，即网站根路径。假设一个网站有一个英文版本，访问域名是中文版域名+"/en"，这时英文站的配置文件中就要修改seajs.root为/en，否则按默认会从根路径加载，加载到的其实是中文站的js文件。

样式重置

样式重置的目的在于清除浏览器的默认样式，带来一个干净的定制空间，因此没有使用Normalize.css这种理念太过“平和”的类库，结合Pc端、移动端、响应式网站的综合需求，在去默认样式的基础上，又额外做了一些事情：

  1. 去掉触屏点击链接的背景色；
  2. 全局设置图片最大宽度100%，开发中会造成的常见问题是图片类效果插件显示异常，或百度地图这类自带图片资源的第三方插件显示异常，只要针对性的添加max-width:none即可；
  3. 对a标签添加0.3s的过渡动画；

Javascript

概述

Js三层结构

通常网站Js脚本可以分为 base + module + app 三层：

base层提供更易用的语法和工具来简化开发，例如jquery，框架另外增加了base.js提供更多基础功能；

module是对特定功能的封装，如轮显效果、视频播放器，日期选择器，表单验证器等等，/static/modules/里除了jquery、base和seajs以外，几乎全部都是已经封装的模块文件，涵盖了绝大部分建站常用效果；

app是基于base和modules实现的具体页面脚本，每个页面都有一个对应的js文件放在项目内app/文件夹中，便于后期维护，app/里还有一个common.js用于管理整站通用代码。

框架重点对base和module这两层做了封装，使开发者能基于这些封装快速、简单的实现页面，关于模块化开发的其他资料可以参考：前端模块化开发的价值 SeajsAPI快速参考

如何使用未模块化的js文件

Q1：如何不破坏模块化开发体验

模块化开发很重要优势是前后端分离，引用js不再是页面底部的一长串script标签，而是入口js中的若干"require"关键字，js功能的增删改都不需要编辑、发布前台模板，使js代码很容易维护，直接用script标签引入第三方插件，那这个页面的维护成本将会很高，也做不到按需加载，理想的方式是可以像引用模块一样的在js中动态载入；

Q2：如何解决文件依赖

一些复杂的插件可能包含多个js文件，甚至他们之间有严格依赖关系，需要满足多个文件依次加载；或者还有配套的css文件，这时又要能同时加载css，而展示类插件多基于jquery，最好还能让他们直接使用框架的jquery对象，并且这些都是按需的；

A：base.getScript()：

经过梳理，我们需要这样一个方法，可以动态载入js文件，这个文件可能依赖jquery、css或其他js，等他们都载入后执行回调，基于以上需求，base.js中封装了base.getScript方法，作为外链第三方js插件的解决方案，【点此了解】。

app

为模板增加一个Js入口

seajs.use可以指定要启动的js文件，如首页底部的：

        
  seajs.use('app/index')

这样首页模板就将执行app/index.js，首页里的所有js效果都要在这里实现。

如何使用模块

要在页面里使用模块实现特定功能，就要先在页面的js里引用模块，引用模块的语法是require(‘模块路径’)，或者用异步加载的方式require.async(‘模块路径’，‘回调’)。

以index.js为例：

   
  /*
   * index.js
   */
  define(function(require, exports, module) {
     var $ = require('jquery');
     var base=require('base');
     var com = require('./common');
     //...
     
  })

index.js里分别引用了jquery，base和common，并将他们的返回值分别赋值。

并不是所有的模块都有返回值，例如common模块多数时候只是存放网站通用代码，导航栏、返回顶部、分享代码之类，很少对外提供方法，如果确定一个模块不需要任何返回值也可以只引用不赋值，例如 require('./common')，大多数jquery插件模块也都没有返回值，因为jquery插件方法都扩展在jquery对象上，可以直接通过$.METHODNAME的方式使用。

异步加载

另一种加载方式require.async是异步加载，异步加载的模块不会提前就位，而是当需要用到时才开始请求模块文件，对于jquery和common这种被其他代码强依赖的模块我们希望尽快加载生效，因此使用同步加载，对于大多数效果类模块则应适度使用异步加载，以提高页面性能，但第一屏内的效果（banner...）应使用同步加载，以提高首屏响应速度。

require.async回来的模块只能在本次请求的回调函数中使用，如果模块有返回值该回调函数将接受该值为参数，以百度分享模块为例，异步请求后回调函数中接收了该模块返回的百度分享方法，并自定义名称为bdshare，然后在函数体内使用bdshare方法： 

      
   //百度分享 
   require.async('bdshare',function(bdshare){
      bdshare({
         bdSize : 16
      });
   });

                  '<img>' +
                  '<div class="ion remove text-danger"></div>' +

  uploader.on( 'uploadProgress', function( file, percentage ) {
      var $li = $( '#'+file.id ),

      if ( !$percent.length ) {
          $percent = $('<p class="progress"><span></span></p>')

下拉框美化插件，作用于原生select标签时会创建一个继承样式的模拟结构，并隐藏原select标签，操作模拟select会将结果同步回原生select，支持disabled和selected特性模拟。

多终端

支持

注意
  1. 在移动端select可以完全自定义样式，所以不需要创建模拟结构，但仍然支持对select的操作回调
配置
data 优先使用此数据初始化，格式见示例
posi 下拉方向，默认"bottom"，可选"top"和"auto"，"auto"会计算下拉框距离屏幕上下两端的尺寸，向大的一边展开。
hideSelected 默认false，是否隐藏已经选中的项
hideDisabled 默认true，是否隐藏不可用的项
act 触发动作，可选 click | mouseenter
hook 样式钩子，用于自定义皮肤，默认default，生成的类名为select-ui-choose-hook和select-ui-options-hook，具体样式写法参考内置样式; 另外内置thin皮肤，适用语言选择类场景
callback(value,text) 选中回调，接收option的value值和text值为参数
示例
               
  <style>
  .demo, button{display: block; width: 200px; margin:10px;}
  </style>
  <form id="test" >
    <!-- 从原生select初始化 -->
    <select class="demo demo-fromdom" name="select1">
      <option selected>--请选择--</option>
      <option value="1">山东</option>
      <option value="2">山西</option>
      <option value="3">河南</option>
    </select>
    <!-- 从数据初始化 -->
    <select class="demo demo-fromdata" name="select2">
      <option disabled>--请选择--</option>
    </select>
    
    <button type="submit">提交</button>
  </form>

slide
描述

幻灯片效果，支持触屏操作，支持PC端图片按需加载（不可与响应式图片同时使用）。

多终端

支持

注意
  1. 自动生成的导航（小圆点）和左右箭头默认display:none且不绑定事件，使用时需先在样式中显示；
  2. 默认导航是带序号的a标签，要自定义a标签内容需要指定导航容器并添加custom属性:<div class="my_navs" custom> 自己写a标签... </div>
配置
wrap 默认"ul"，指定包裹元素
cell 默认"li"，指定切换元素
effect 切换效果，默认‘slide’，可选slide\fade\toggle
direction 默认"x"，slide效果滚动方向，可选 x \ y
animate 默认"ease"，css3动画曲线
duration 默认200，切换动画时长(ms)
start 起始帧，默认0，即从第一张开始
auto 默认true，自动播放
pause 默认true，控制是否鼠标悬停时暂停播放，还可以传入选择器将作为暂停按钮，点击暂停，按钮将被添加'pause'类
interval 默认5000，自动播放间隔(ms)
act 默认‘click’，导航按钮触发事件 mouseenter | click
prevHtml 左箭头html
nextHtml 右箭头html
prev 指定左箭头，不指定将自动插入'.arrs .arr_prev'元素
next 指定右箭头，不指定将自动插入'.arrs .arr_next'元素
navs 指定自定义导航，不指定将自动添加'.slide_nav'元素或使用效果元素内已存在的".slide_nav"元素;
imgattr 默认"slide-src"，按需加载图片的地址标签，注意不能是data-src，因为与响应式图片方法冲突
handletouch 默认false，是否劫持触屏事件
callback 回调，@param (效果元素，切换元素，当前帧序号)
ext 扩展，@param (效果元素，切换元素，总帧数)

tip
描述

提示框插件，语法： $handle.tip($target,option); ，$target是被弹出内容，支持文本，html，jQuery元素（将清除默认样式）

多终端

支持

配置
act 默认‘mouseenter’，触发动作 mouseenter | click
place 弹出元素相对于$handle的显示位置，默认"left-center"，规则"第一轴向-第二轴向-[定位模式]"，第一轴向可选"left | right | top | bottom"，第二轴向除四个方位外还可以选择"center"，定位模式非必填，可选"in"，"in"定位模式表示$target将覆盖$handle
title 标题文字，默认false，不显示标题栏
hasarr 默认true，提示框有无箭头
offset 提示框与元素间距，默认0
type 默认'auto'，弹出文本内容自带边框，DOM元素不带边框，可选'content'，所有元素均自带边框弹出
modal 是否带半透明背景，默认false
opacity 模态背景透明度，默认0.5
hook class钩子，将添加到弹出元素上，默认无
show 立即显示，默认false，若为true将忽略act事件，要用返回值中的hide()方法关闭
onshow 显示回调，接收当前对象为参数
onhide 关闭回调
返回值
obj.hide() 关闭当前tip实例
示例
         
  <style>
  /*css*/
  .demo a{margin: 60px;}
  </style>
         
         
  <!--html-->

  </div>

base.getType([callback])

获取当前屏幕类型，回调函数可选，当无回调时返回字符串('Pc','Pad','Mobile')，有回调时回调接受返回字符串为参数：

         
   var base=require('base');
   
   //无回调
   var _type=base.getType();      //"Pc" || "Pad" || "Mobile"
   
   //有回调
   base.getType(function(type){
      alert(type);                //"Pc" || "Pad" || "Mobile"
   })
base.getOrient([callback])

获取移动设备屏幕方向，回调函数可选，当无回调时返回字符串('Heng','Shu')，有回调时回调函数将持续监听屏幕方向变化，并接受屏幕方向为参数：

            
   var base=require('base');
   
   //无回调
   var _orient=base.getOrient();// "Heng" || "Shu"
   
   //有回调
   base.getOrient(function(orient){
      alert(orient);// 每次旋转屏幕都将alert "Heng" 或 "Shu"
   })

解决方案

多终端

多终端网站开发除了css媒体查询以外，至少还要具备三项功能：响应式栅格系统、终端识别，响应图片。

响应式栅格系统使页面布局可以随终端自动调整，细节见【布局】；终端识别使js开发可以为每个终端做单独处理；响应图片使移动设备可以使用单独定制的小尺寸图片，为移动端提速。

多终端项目模板的实现方案是：responsive.css中定义了两个响应点以区分pc、pad、mobile三种设备，style.css为pc设备提供样式并作为多终端基础样式，另外通过媒体查询“@import”进Pad.css和Mobile.css叠加在style.css上，分别为中屏和小屏设备提供样式，从而实现css的响应；css响应的同时会为body加上特殊的隐形标记，js的设备识别就是通过检测该标记实现的，这种方式的优点是，css和js的响应都来自responsive.css的驱动。

responsive.css中的实现：

     
/* import responsive stylesheet */
@import url(./Pad.css) screen and (max-device-width: 1024px);
@import url(./Mobile.css) screen and (max-device-width: 640px);
...

为进一步细分设备，在Pad.css和Mobile.css中又分别增加了768和320两个响应点，这样在css上可以做到五级划分：

{Pc} > 1024 > {PadLg} > 768 > {PadSm} > 640 > {MobileLg} > 320 > {MobileSm}

js检测和栅格系统则都是三级划分：

{Pc} > 1024 > {Pad} > 640 > {Mobile}

这种样式组织方式有两个弊端：一，采用pc优先原则而非响应式开发提倡的移动优先原则，会给移动端带来相对多的加载压力；二、每个版本一个独立样式文件通过@import载入相比单文件组织所有样式会增加两次请求。

最终采用这种方式是考虑到：一、为保证框架的通用性，必须优先满足普通pc站的兼容性要求，因此只能pc优先，移动端的性能优化在【解决方案】中有相应的功能实现；二、每个设备一个样式文件更易于开发，无须管理庞杂的多终端样式，还能简化配置，只要在responsive.css中配置一次，多版本样式加载、判断屏幕类型、图片响应都会通过这一次配置实现。

图片响应、设备识别等辅助方法都在【js-base】中有详细说明。

<textarea>标签在页面中具有延迟渲染的特性，因此是一个天然的缓存容器，只要将自身隐藏，便实现了html5 template标签的特性，唯一的问题是使用<textarea>暂存的代码中不能包含另一个<textarea>，否则会造成暂存标签意外闭合，这时也可以选择<script type="text/template">标签代替，它和textarea标签的作用是等价的，并且script标签自身就是不可见的，无须手动隐藏：

       
     <!--textarea标签暂存页面片段-->

        <textarea class="hide">

          <a href="#">暂存的页面片段</a>
          <a href="#">暂存的页面片段</a>
        </textarea>
     </div>
     
     
     <!--script标签暂存页面片段-->
     <div class="demo topush">
        <script type="text/template">
          <a href="#">暂存的页面片段</a>
          <a href="#">暂存的页面片段</a>
          <a href="#">暂存的页面片段</a>
        </script>
     </div>

base.topush(DOM,callback)方法专门用来取出暂存标签中的html片段（或发起Ajax）插入到外层元素中，两个参数分别是指定外层元素和取出后的回调函数，均为可选，DOM默认.topush。

加载后的dom会添加pushed类，用来标记暂存代码已经取出。

示例：

      
   <!--发起Ajax的例子-->
   <div class="demo topush">
      <textarea class="hide" data-url="/test/ajax.php"> </textarea>
   </div>
      
      
   //无参数调用，显示所有".topush"元素中的暂存代码
   base.topush();

   base.topush(function(e){           
      console.log(e.attr('class')) //"topush ..."
   });
   
   //指定渲染目标                 
   base.topush('.demo');

base.scanpush()

base.scanpush()方法会扫描页面上具有延迟渲染类名的元素，并按规则对其渲染。

预置的四个按需渲染类是：.PcPush,.UnpcPush,.MobilePush,.UnmobilePush, 分别代表“只有Pc渲染”，“只有Pc不渲染”，“只有手机渲染”，“只有手机不渲染”,渲染完成后会触发该元素的渲染类同名事件，用$(DOM).on([PcPush/UnpcPush/MobilePush/UnmobilePush])可以捕捉到，事件也会冒泡到body上。

 </div>
 
 <div class="UnmobilePush">
    <textarea>
       除了手机都显示
    </textarea>
 </div>
               
               
  //js

  $('body').on('PcPush',function(){
    console.log( 'PcPush已经被渲染了' )

使用

预加载技术有一定特殊性，普通模块加载方式将失效，需要使用优化后的pjax专用模板，开发中基本保持了原有模块化开发体验，主要存在以下三点细节上的不同：

区别1：页面底部引用js时增加回调并执行返回函数：
      
      //页面引用js的写法
      seajs.use('js/index_pjax', function(run) {
        run();
      })
区别2：在页面js的返回函数里写业务代码：
      
  define(function(require) {
    var $ = require('jquery');
    var base = require('base');
    var com = require('./common_pjax'); //注意这里必须引用common_pjax
    //模块js的写法
    return function(){ 
      //页面代码里执行common_pjax
      com.init(); 
      //以下就是你得业务代码啦
      require('box'); 
      $.box.msg('hello p1!');

    }
  })
区别3：原来common.js里的码要写在common_pjax.js的init方法里：
      
  //common_pjax
  module.exports = {
      init:function(){
        //整站通用代码

      }
