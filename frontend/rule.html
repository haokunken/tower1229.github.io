基本规范

关于

《规范》用于指导基于模块化框架和组件化框架的开发，以避免异常和便于维护。

HTML

  1. 编码utf-8；
  2. H5文档格式；
  3. 模板后缀.htm；
  4. 模板以“栏目_功能”格式命名，如“news_list.htm”、“pro_detail.htm”；

CSS

  1. 编码utf-8；
  2. 使用class选择器；
  3. 选择器命名务必表意清晰，建议“栏目_表现形式”格式，如“.news_list”、“.pro_detail”；
  4. z-index最高层级99；
  5. 私有后代选择器命名以下划线（_）开头，如“.job ._item”、“.job ._cell”；

javascript

  1. 编码utf-8；
  2. 模板有唯一的js入口文件，模板js放在项目下app/文件夹，与模板同名，统一引用common.js组织公用脚本；
  3. 非模块化文件放在项目下lib/文件夹，单独在模板头部/底部普通引入；

插件开发

  1. 插件库是框架根目录下modules/，插件只有一个js文件或只依赖一个css文件可以直接放在modules/根目录，包含更多文件（如图片）的插件放在modules/插件同名文件夹/下；
  2. 文件顶部统一注释：文件名、GUN风格版本号、更新内容、更新日期；
  3. 书写格式视模块加载器（seajs\mod）而定，依赖文件尽量与js文件同名，css文件顶部同样需要注释css文件名、版本号、更新内容、更新日期；
  4. 以小粒度、潜封装为原则，每个插件只实现一个特定功能（功能粒度小），实现上保留最大的结构开放性和样式扩展性（封装层次潜）；

组件开发

  1. 组件库是框架根目录下widget/，组件按“类目/序号/”的结构存放，如“form/1/”；
  2. 组件由5个文件组成：“temp.htm”，“style.css”，“script.js”，“config.json”，“album.jpg”，前三个文件均使用etpl模板渲染
  3. 组件css命名空间规则为“.widget-类目-序号”，如“.widget-form-5”，组件内全部继承命名空间，防止样式冲突
  4. 使用按需渲染功能的组件js中要先调用base.scanpush()方法，否则无法演示
